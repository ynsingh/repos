Table Asks for Information to be filled about Remote Institue who is interested to buy or sell course.
Following are the Table Details :
  1. Course ID -- Course Id of course of Instructor in Remote institute
  2. Instructor Name -- Name of Instructor(Login Name) in Remote institute
  3. Institute I.P Address -- IP address of Remote institute
  4. Institute Name -- Institute Name of Remote institue
  5. Secret Key -- Secret Key to be agreed by both party
  6. Transaction -- Buy/Sell
  7. Expiry Date -- Expiry date upto which course can be accessed

How To Use Remote Course

    Suppose you (Instructor of Course) want to sell/buy your course content to instrutor of another Institute using Brihaspati LMS.
  1. You should fill the given table and press submit.Now you will be shown your entry (highlighted among other entries).If you feel that the entry is not correct ,click on update link.Correct the entries and press Update.Now again you will be shown your entry.
  2. Next you have to check if the remote instiute has registered you in his brihaspati server or not .To verify this click on Check-Remote-Registration link.If you are registered and you are buyer than you click on Get - Course - Content link .
  3. If you get message that course was send succesfully click on link Course Management than on Course Content.Now you will see that requested Remote Course Id contents are avalable for use.
  4. Your courses are now in unpublished area,you will have to publish them.Your Remote course Contents will be updated an daily basis(24 hours).Once your suscription has expired you have to re-buy else you will lose the links to remote courses.

Known limitations of Remote Courses

  1. Remote Courses works only on port 8080
  2. Only those courses which are visible /accessible can be selled as Remote course.
  3. Those courses which are obtained from othes via permission or remote course (BUY) cannot be selled
  4. Until you register IP ADDRESS and Course ID of Remote user with local server you cannot sell/buy any course
  5. If the remote server is not running you cannot access remote course
  6. If the port 12345 is not alive remote courses access can not be done
